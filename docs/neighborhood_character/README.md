# Neighborhood Character

This folder defines how Stoop thinks about neighborhood character across apps,
data products, and future scoring systems.

The goal is to create a shared language for describing what a neighborhood is
like, what makes it useful for different use cases, and how those ideas should
eventually map to data models and app outputs.

## Scope

Neighborhood character is a broad framing layer that supports two related but
distinct product questions:

- **Stoop Explore**: Where should I spend a day?
- **Stoop Search**: Is this somewhere I would like to live?

These are not separate concepts from scratch. They are two applied views of a
shared neighborhood character model.

## Initial document set

- [framework.md](framework.md): core definition, source layers, product views,

- [signal_inventory.md](signal_inventory.md): lane-first inventory of available

## Working principles

- Start descriptive before becoming evaluative.
- Use demographics as background context, not the dominant story.
- Keep cultural and civic signals distinct when they serve different use cases.
- Prefer natural-language outputs backed by explainable data inputs.
- Start by indexing neighborhoods relative to NYC, while designing for future
  market expansion.
